## apb 0.6.0: hermes agent support

Hermes Agent (Nous Research's `hermes` CLI) joins claude, codex, agy, opencode, and pi as the sixth known executor agent: detection, profiles, and one-shot execution all work out of the box, no custom `agents:` entry required.

### Detection and invocation

The built-in probe looks for a `hermes` binary on `PATH`, reads its `--version` output, and classifies it as an aggregator agent like agy, opencode, and pi. An auth hint is reported when `<home>/.hermes/.env` exists; apb never reads the file's contents, it only checks presence.

A node bound to a hermes profile runs `hermes -z "<prompt>" -m "<model>"` in the node's working directory, the same headless one-shot pattern as codex `exec` and opencode `run`. The SOUL (role prompt) is delivered as a prefix on the prompt itself, since hermes has no separate system-prompt flag. Hermes' `-z` mode prints only the final response text to stdout, which becomes the node's output.

### Model selection

Model selection is explicit: the profile's `model` field is passed straight through as `-m <model>`, and hermes auto-detects the provider from the model name itself, apb makes no provider choice on hermes' behalf. apb ships no curated model inventory for hermes yet (see Known limitations), so `profile_howto`'s advisory models table currently has no hermes-specific rows; pick a model hermes already recognizes and pass it directly.

### Trust posture

Hermes' `-z` one-shot mode auto-approves tool use for the run and loads `AGENTS.md`, rules, memory, and the user's configured toolsets from the working directory and user config. This is the same trust posture apb already accepts for codex `exec` and opencode `run`: no interactive approval prompt exists in one-shot mode, so binding a node to hermes means the run's effective permissions are whatever hermes' own config and toolsets allow. Users who need a stripped-down hermes for apb runs can bind a profile to a wrapper script or constrain toolsets through hermes' own configuration.

### Known limitations

No hermes model inventory: `ModelsSource::None` for the hermes probe, so no automatic model list is queried or asserted. No ACP transport: hermes is invoked headless like the other aggregators, not through apb's `acp` transport (which is Claude Code stream-json today). No session resume mapping: hermes' own `--resume`/`--continue` flags are not wired to apb's run resume. No usage-file cost capture: hermes' `--usage-file` output is not read into run events.
