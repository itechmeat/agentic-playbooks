## apb 0.3.0: official connectors, wave 1

This release ships the first four official connectors and the engine
capabilities they rely on. Connectors are embedded in the binary: whoever
has `apb` has the matching connector versions.

### Official connectors

Install with `apb connector install <name>`; installing from the binary
records trust in the same action.

- **github** - issues, pull requests, releases, workflow dispatch, checks
  and statuses over the REST API. Keys-free setup on developer machines:
  the account config can source the token straight from the GitHub CLI
  with `token: "{{cmd:gh auth token}}"`.
- **telegram** - bot messaging: send and edit messages, inspect chats,
  long-poll updates, answer callback queries.
- **smtp** - transactional email over SMTP with STARTTLS, plus a `verify`
  healthcheck that probes the connection without sending.
- **sentry** - issue search and triage, releases and deploys, with
  cursor pagination through the call result's `link` field.

### Connector format and engine

- Command-sourced secrets: a secret account field may reference a command
  (`{{cmd:...}}`) instead of an env variable; resolved in memory at call
  time, never stored, covered by the account trust digest.
- New `path` auth kind with the reserved `{{auth}}` URL placeholder
  (required for the Telegram Bot API, which carries the token in the URL
  path); the token never reaches logs.
- New built-in `smtp` function kind executed natively by the engine:
  STARTTLS, multipart messages, taxonomy-mapped errors. Manifests stay
  declarative; connectors still execute no code.
- Response shaping: functions may declare `response_pick` so agents get a
  trimmed projection by default; `apb connector call --full` returns the
  complete body for debugging.
- Per-function `headers`, a default `User-Agent`, authored `examples`
  rendered into the agent instruction, and the response `Link` header
  surfaced in the call result.

### Tooling

- `apb connector install <name>` and `install --from-dir <path>`; the
  connector list shows embedded-but-not-installed connectors and version
  drift after a binary upgrade.
- Declarative contract tests: each connector ships a `tests.yaml`, run
  fully offline by `apb connector test`; CI keeps every official
  connector installable and correct.
- Dashboard playground on the connector detail page: call any function
  manually with a form generated from its schema, dry-run by default.
- Demo playbooks (`sentry-triage`, `release-announce`) exercising the
  connectors end to end.

### Release process

- The GitHub release body now comes from `docs/release-notes/<tag>.md`,
  prepared and reviewed in the release pull request; tagging without
  notes fails the release workflow early.
