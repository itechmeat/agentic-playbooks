## apb 0.10.0: official connectors, wave 3

Five new official connectors ship in this release: `slack`, `zulip`, `gitlab`, `youtrack`, and `discord`, bringing the official set to eleven. All five are pure HTTP connector manifests, installable with `apb connector install <name>` and covered by the same three-tier testing as the earlier waves (per-manifest contract tests, engine render tests against a mock HTTP server, and env-gated live smoke tests). The wave also adds two small generic engine extensions the new services required: `body_form` for form-encoded request bodies and `error_when` for reclassifying envelope-style errors hidden inside 2xx responses. Full design: `docs/superpowers/specs/2026-07-22-official-connectors-wave-3-design.md`.

### The five connectors

**gitlab** reaches parity with the github connector core, mapped to GitLab idioms: issues (list, get, create, update, comment), merge requests (list, get, create, merge, approve, comment), releases (create, get by tag), and pipelines (list, get, list jobs, trigger). Auth is a personal access token in the `PRIVATE-TOKEN` header; every project-scoped function takes the project as an argument (numeric id or `group/project` path, percent-encoded by the engine), so one account serves any number of projects on gitlab.com or a self-hosted instance. Label edits go through `update_issue` (`labels`, `add_labels`, `remove_labels`) following the native API rather than imitating github function names. `trigger_pipeline` is an ordinary grant-gated write: grant it only to playbooks meant to start CI, not observe it.

**youtrack** covers issue search and CRUD, comments, project listing, and `apply_command`, YouTrack's native command syntax for state changes, tagging, assignment, and custom fields in one call. Auth is a permanent token as a Bearer header. Every read function bakes an explicit `fields=` projection matching its `response_pick`, so responses carry exactly the projected fields. `search_issues` uses YouTrack query syntax and pages with `$skip`/`$top`. `apply_command` can change almost anything on an issue, so PUBLIC.md states plainly what a grant including it allows.

**zulip** reads streams, topics, and messages and posts stream and direct messages. Auth is HTTP Basic with the account email and API key, and writes post `application/x-www-form-urlencoded` bodies through the new `body_form` extension, matching Zulip's native write contract. A thread reply in Zulip is a stream message to the same topic, so `send_stream_message` covers it. The manifest sticks to long-stable endpoints so older self-hosted servers work.

**discord** lists guild channels, reads recent channel and thread messages, and posts messages and replies with a bot token (`Bot` header prefix). Threads are channels, so reading a thread is `get_messages` with the thread's channel id. The connector is REST-only; the gateway-only message-content privileged intent is not needed. PUBLIC.md documents the exact bot permissions per function and warns about Discord's aggressive per-route rate limits.

**slack** lists channels, reads channel history and threads, and posts messages and thread replies with a bot token. Slack reports API failures as HTTP 200 with `"ok": false`, which the manifest handles with the new `error_when` block (below), so a failed Slack call is a real service error carrying Slack's `error` string (`missing_scope`, `channel_not_found`) instead of a false success. PUBLIC.md documents the granular bot scopes per function group and the channel-invitation requirement. `send_message` and `reply_in_thread` are separate functions over the same endpoint so a grant can allow thread replies without allowing new top-level posts.

### Engine extension: body_form

An HTTP function can now declare `body_form`, a string-to-template map rendered as an `application/x-www-form-urlencoded` request body with percent-encoded keys and values. It is mutually exclusive with `body` (declaring both is a validation error) and allowed only on methods that carry bodies. The wave-2 single-placeholder semantics apply: a value that is exactly one `{{args.<field>}}` placeholder renders the typed argument, and an absent optional argument drops the whole pair from the encoded body. Values follow function-body template rules (`account.*` and `args.*` only, never `secret.*`). Connector contract tests gain `expect.body_form_contains`, a subset match over the decoded pairs mirroring `body_contains`.

### Engine extension: error_when

A connector can now declare, at the manifest level, when a 2xx response is actually an error: `error_when: { path, equals, message_path }`. After a 2xx response the engine evaluates `path` against the parsed JSON body; on a match the call maps to a `service` error whose message is the string at `message_path`, so retries, fallbacks, and policy gates react to envelope-style failures exactly like HTTP-level ones. Non-2xx statuses keep the existing status-table mapping, `response_pick` is not applied to a reclassified response, and the extracted message passes through the same redaction as any other body text. Mock, smtp, and imap functions ignore the block. Slack is the motivating consumer; the block is generic and reusable by any future API with the same convention.

### Demo playbook and docs

`examples/playbooks/release-heartbeat.yaml` is the wave's demo: check the latest gitlab pipeline on a ref (`list_pipelines`, `get_pipeline`), have the agent summarize the state, post the summary to a Slack channel, and file a youtrack issue when the pipeline failed. Grant allowlists name exactly the functions used; `trigger_pipeline` is deliberately absent, modeling the restraint pattern for consequential writes. It validates in CI against fake accounts like the earlier demos. `docs/CONNECTORS.md` gains a setup section per service covering token creation, required scopes or permissions, and provider-specific notes (Slack scope granularity and channel membership, Discord bot permissions and rate limits, GitLab project addressing, YouTrack command syntax, Zulip self-hosted compatibility).

### Testing

The manifest CI gate picks up all five connectors automatically (parse, digest, PUBLIC.md frontmatter, strict schemas, read-only healthcheck, examples, at least one contract case per function). Engine render tests drive each checked-in manifest through the real call path against a mock HTTP server: gitlab's `PRIVATE-TOKEN` header and page pagination, youtrack's `fields=` projections, zulip's Basic auth and exact percent-encoded form bodies on the wire, discord's `Bot` prefix, and slack's Bearer header, cursor projection, and `ok: false` reclassification. Env-gated live smoke tests (`APB_LIVE_TEST_<SERVICE>=1`, never run in CI) probe each connector's healthcheck plus one read-only function against a real account.

### Supervised-run hardening (issue 45)

This release also lands the full fix wave for the fourteen findings of issue 45, collected from real supervised runs. The engine gains a marker form of `success_check` (`success_check: { marker: "..." }`) that rejects an agent's premature success report when the completion marker is missing from the node output, plus a V33 validation code for misplaced or empty checks and an authoring-guide warning describing the premature-success class and how to design around it. Run status surfaces became truthful: a supervised run waiting on a supervisor decision now reports a first-class `pending_supervisor` state (node, trigger, and the available options) in `run_status`, `apb doctor`, and the web run view; a live open attempt no longer folds to `interrupted`; and a child run driven by its parent records `driven_by` instead of a stale foreign `driver.pid`, so doctor stops failing healthy children.

The supervisor channel now actually delivers: notes posted with `supervisor_context_append` are injected into every subsequent agent attempt's prompt as a delimited supervisor-notes block, and a child run's wakes are mirrored into the parent's supervisor channel so `supervisor_wait_event` sees child questions and anomalies. Resume after environment drift is wired end to end: `apb resume --allow-environment-drift` and the matching MCP `run_resume` parameter reach the engine override, the drift preflight runs before detaching so a refused resume fails inline, and a detached driver that dies at startup records a `RunError` event visible in status and doctor. A new capability-gated `supervisor_rebind_profile` tool gives supervisors a sanctioned escape hatch when a pinned executor is down: the new profile bundle passes the same trust gate as run start, the rebind is journaled into an overlay while the original manifest stays intact, and the next retry runs under the new binding. Patch validation now scopes the executed-node immutability check to the preserved prefix that will not re-run under `continue_from`, and a new V34 validation code rejects contradictory duplicate edges (identical routing condition, different targets). Node timeouts kill the whole attempt process tree the same way cancellation does. Two CLI paper cuts are fixed: the connector `--account` error now names the granted accounts (and account acceptance stays strictly bound to the grant), and a failed `apb dashboard` bind names the process holding the port with a hint instead of a bare error.

Two design notes accepted consciously in this wave: the `rebind` capability joins the default supervisor capability set, consistent with the documented default-equals-all-implemented model (playbooks that pin an explicit capability list are unaffected), and the `continue_from` re-run set is computed structurally (condition-blind), which can classify a conditionally-skipped executed node as re-runnable; both are noted in the code and docs.

### Known limitations

No OAuth flows inside apb: tokens come from account configs, optionally via the `{{cmd:...}}` helper. No websocket or gateway transports (Discord gateway, Slack Socket Mode, Zulip event queues); reading recent history by polling inside a node is the interim answer. No file or attachment upload or download in any of the five connectors. Pagination stays a per-manifest idiom (cursor, page, `$skip`/`$top`, anchor, before-id); a shared abstraction remains a future decision. Discord's global rate-limit behavior beyond the status-table 429 mapping is not special-cased this wave.
